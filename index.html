HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS is all about style, baby! (Colors, shadows, margins, etc)
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • If I get stuck in a cd, I can go back by using cd ~
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Git Flow: add, commit & push to the branch, open pull request, merge PR with main branch, close issue
  • Step 1: Navigate to correct folder, then use git add -A to stage changes
  • Step 2: Add commit message with git commit -m "your message here”
  • Step 3: Pull the base branch main into your new branch with git pull origin main
  • Step 4: To complete Git Flow, use git push origin NameOfFile
  • Step 5: Changes committed, go to GitHub branch, then "Pull Requests", New Pull Request (Make sure base is main and new branch as comparison)
  • Step 6: Click "Create Pull Request", give title and desc., Merge Pull Request
  • Step 7: Close GitHub Issue
  • To create feature branch, use git checkout -b feature/action-program

JavaScript

  • JavaScript makes a static HTML/CSS webpage and makes it interactive
  • A variable is simply a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • An array is a single variable that is used to hold a group (normally related) of data.
  • The first item in any array has an index of 0, not 1.
  • Function = set of instructions that tells the comp how to do a certain task.
  • A function's name is followed by parentheses ()
  • To add to console: console.log("what you want logged in console")
  • “Commented out” (making it inactive) by adding /* before the block of code and */ after.
